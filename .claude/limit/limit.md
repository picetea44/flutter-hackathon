# 프로젝트 제약 사항 (Code Constraints)

## 공통 규칙

1. 모든 파일은 최대 100줄을 넘지 않는다.
2. 모든 함수/메서드는 최대 10줄로 제한한다. (Java 기준)
3. 파라미터는 최대 4개까지만 허용한다.
4. 모든 원시 값과 문자열은 전용 객체로 래핑한다.
5. 줄여 쓰기(축약)를 금지한다.

## Java/Spring 상세

1. Indent Depth는 1까지만 허용한다.
2. `else`, `switch/case` 사용을 금지한다. (Early Return 권장)
3. 3항 연산자를 사용하지 않는다.
4. 배열 대신 컬렉션을 사용하며, 모든 컬렉션은 일급 컬렉션으로 구현한다.
5. 모든 엔티티와 도메인 로직은 TDD로 검증한다.
6. UI 로직(Input/ResultView)과 비즈니스 로직을 엄격히 분리한다.

## React 상세

1. 컴포넌트는 단일 기능만 수행하며 50줄 이내로 유지한다.
2. 로직(Hook)과 표현(Component)을 분리한다.
3. 비즈니스 로직은 Pure Function으로 추출하여 테스트한다.
